Celery — это распределённая система для обработки задач в фоновом режиме (background tasks).
Она позволяет отправлять задачи на выполнение в отдельных процессах или на других серверах, не блокируя основное приложение.
Это особенно полезно для длительных или ресурсоемких операций, таких как отправка электронной почты, обработка изображений, выполнение вычислений и др.
Celery используется для решения проблем, связанных с выполнением длительных или ресурсоемких задач в веб-приложениях,
не блокируя при этом основной поток выполнения и обеспечивая высокую доступность и масштабируемость.

Celery — это неблокирующая очередь задач, работающая в распределенной системе.
Он может управлять асинхронными фоновыми процессами, которые огромны и требуют большой нагрузки на процессор.

Broker (Брокер): Сервер, который хранит задачи в очереди (queue).
 Celery поддерживает несколько брокерских систем, таких как RabbitMQ, Redis, Amazon SQS и другие.
  Брокер — сердце системы, он обеспечивает надежное хранение и доставку задач.

Worker (Рабочий): Процесс (или процессы), которые извлекают задачи из очереди и выполняют их.
 Можно запускать несколько рабочих процессов для обработки задач параллельно.
 Рабочие могут быть размещены на одном сервере или распределены между несколькими серверами для масштабирования.

Task (Задача): Функция Python, которая должна быть выполнена в фоновом режиме.
 Задачи определяются с помощью декоратора  @app.task.

FastAPI → (1) → Redis → (2) → Celery Worker → (3) → Redis → (4) → FastAPI
  (ставит задачу)   (очередь)   (выполняет)    (результат)  (читает)

Шаг Откуда  Куда    Что происходит
1   FastAPI        →    Redis Задача сохраняется в очередь
2   Redis          →    Celery Worker Воркер забирает задачу
3   Celery Worker  → Redis Результат сохраняется
4   Redis         → FastAPI   Приложение может проверить результат

Celery использует архитектуру “производитель-потребитель” (producer-consumer) для обработки задач.
 Два основных компонента этой архитектуры — это брокер (broker) и обработчики (workers).

Брокер — это сервер, который выступает в качестве посредника между производителем задач (вашим приложением)
и потребителями задач (обработчиками). Он отвечает за хранение задач в очередях (queues) и их распределение между обработчиками.

Популярные брокеры для Celery:

RabbitMQ: Популярная система для обработки сообщений. Известна своей надежностью и производительностью.

Redis: Быстрое хранилище данных в памяти. Идеально подходит для высокопроизводительных систем обработки задач.

Обработчики — это процессы (или процессы на разных машинах), которые извлекают задачи из очередей брокера и выполняют их.

Приложение отправляет задачи в брокер. Обработчики подключаются к брокеру и извлекают задачи из очередей. П
осле выполнения задачи рабочий может отправить результат в брокер или систему хранения результатов.

В целом, брокер и рабочие являются ключевыми компонентами Celery, которые обеспечивают надежную и масштабируемую обработку задач в фоновом режиме.
 Правильный выбор брокера и оптимальная настройка обработчиков — важные факторы для эффективного использования Celery.

start redis: redis-server or redis-server /opt/homebrew/etc/redis.conf

Чтобы просмотреть первые 100 задач в очереди в Redis
-redis-cli lrange celery 0 100

При запуске Celery по умолчанию создается 1 обработчик(процесс).
-celery -A main.celery worker
-celery -A main.celery worker --loglevel=info


Flower - это инструмент мониторинга Celery, который наблюдает и контролирует выполнение всех задач,
генерируя аудит в реальном времени на веб-платформе
Flower — это простой и удобный веб-интерфейс для мониторинга Celery в реальном времени.
Он позволяет отслеживать выполнение задач, просматривать состояние воркеров и очередей, управлять задачами (например отменять их),
 а также анализировать статистику.

-pip install flower




В данном случае в разных консолях нужно запустить:
redis-server ---Запуск сервера Redis
-celery -A main.celery worker --loglevel=info ---Запуск процесса
-celery -A main.celery beat --loglevel=info ---Периодические задачи
-uvicorn main:app --reload --port 8080 ---Запуск сервера uvicorn - fastapi
-celery -A main.celery flower ---инструмент мониторинга Celery
